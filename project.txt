# 🧠 🔐 OVERALL IDEA

Your system is:

> **Continuous Authentication using Behavioural Biometrics**

👉 Instead of checking user only at login,
👉 it **continuously verifies identity** using behaviour.

---

# ⚙️ FULL INTRUSION DETECTION FLOW

---

# 1️⃣ DATA COLLECTION (FRONTEND)

On **details + interaction pages**:

* ⌨️ Keystrokes captured

  ```json
  { key: "a", time: 123456 }
  ```
* 🖱️ Mouse movements captured

  ```json
  { x: 200, y: 150, time: 123456 }
  ```

👉 Sent every few seconds to backend:

```plaintext
/capture API
```

---

# 2️⃣ FEATURE EXTRACTION

Inside `feature_extractor.py`

We convert raw data → meaningful behaviour:

### ⌨️ Typing features:

* **key_mean** → average typing speed
* **key_std** → variation in typing
* **digraph timing** → time between key pairs (like "th", "he")

👉 This captures **typing rhythm**

---

### 🖱️ Mouse features:

* movement speed
* direction consistency

---

👉 Final feature vector:

```plaintext
{
  key_mean,
  key_std,
  mouse_mean,
  digraph patterns
}
```

---

# 3️⃣ LEARNING PHASE

```plaintext
First 12 samples → learning
```

👉 System builds **user profile**

Profile = average behaviour:

```plaintext
profile = average(features)
```

👉 This becomes:

> 🧠 “normal behaviour of the user”

---

# 4️⃣ CONTINUOUS MONITORING

After learning:

Every new input:

```plaintext
current behaviour → extracted
```

System compares:

```plaintext
profile vs current
```

---

# 5️⃣ SIMILARITY CALCULATION

We compute:

```plaintext
difference = how far current is from profile
```

Includes:

* typing speed difference
* typing variation
* mouse behaviour
* digraph timing

---

Then:

```plaintext
score = 1 - difference
```

👉 So:

* score ≈ 1 → very similar ✅
* score ≈ 0 → very different ❌

---

# 6️⃣ ADAPTIVE THRESHOLD

Instead of fixed value:

```plaintext
threshold = average(user behaviour) - margin
```

👉 Each user gets:

> 🎯 personalised threshold

---

# 7️⃣ DECISION LOGIC

Final check:

```plaintext
if score >= (threshold - tolerance):
    genuine
else:
    intruder
```

👉 Tolerance avoids false alarms

---

# 8️⃣ 3-STRIKE INTRUSION CONTROL

Very important 🔥

Instead of instant logout:

```plaintext
Intruder → count = 1
Intruder → count = 2
Intruder → count = 3 → LOGOUT
```

👉 Why?

* prevents false positives
* increases reliability

---

# 9️⃣ COOLDOWN MECHANISM

Even after detection:

```plaintext
alert sent only after cooldown time
```

👉 Prevents:

* spam alerts
* repeated triggers

---

# 🔁 COMPLETE FLOW SUMMARY

```plaintext
User logs in
    ↓
System collects behaviour (typing + mouse)
    ↓
Feature extraction
    ↓
Learning phase (build profile)
    ↓
Continuous monitoring
    ↓
Similarity score calculated
    ↓
Compare with adaptive threshold
    ↓
If abnormal → intruder count++
    ↓
If 3 consecutive → logout + alert
```
